**Step 8 -- .env setup + API key prompts (msg pair 8 of 8, INST-07, INST-08, D-01 through D-04):**
```
msg_info "Configuring Environment"
if [[ ! -f /opt/openmontage/.env ]]; then
  cp /opt/openmontage/.env.example /opt/openmontage/.env

  read -rp "Enter FAL_KEY (or press Enter to skip): " FAL_KEY_INPUT
  if [[ -n "${FAL_KEY_INPUT}" ]]; then
    sed -i "s|^FAL_KEY=.*|FAL_KEY=${FAL_KEY_INPUT}|" /opt/openmontage/.env
  else
    sed -i "s|^FAL_KEY=.*|# FAL_KEY=your-key-here|" /opt/openmontage/.env
  fi

  read -rp "Enter ELEVENLABS_API_KEY (or press Enter to skip): " ELEVENLABS_INPUT
  if [[ -n "${ELEVENLABS_INPUT}" ]]; then
    sed -i "s|^ELEVENLABS_API_KEY=.*|ELEVENLABS_API_KEY=${ELEVENLABS_INPUT}|" /opt/openmontage/.env
  else
    sed -i "s|^ELEVENLABS_API_KEY=.*|# ELEVENLABS_API_KEY=your-key-here|" /opt/openmontage/.env
  fi

  read -rp "Enter OPENAI_API_KEY (or press Enter to skip): " OPENAI_INPUT
  if [[ -n "${OPENAI_INPUT}" ]]; then
    sed -i "s|^OPENAI_API_KEY=.*|OPENAI_API_KEY=${OPENAI_INPUT}|" /opt/openmontage/.env
  else
    sed -i "s|^OPENAI_API_KEY=.*|# OPENAI_API_KEY=your-key-here|" /opt/openmontage/.env
  fi
fi
msg_ok "Configured Environment"
```

Use `sed -i` to modify the existing lines from `.env.example` in-place rather than appending (avoids duplicate keys). The `.env.example` already contains these key names as `KEY=` lines.

The entire API key prompt block is inside the `if [[ ! -f .env ]]` guard -- on re-install/update, .env is preserved and prompts are skipped (per D-06 and CLAUDE.md anti-pattern).

**Mandatory closing sequence (exact order):**

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> .env file | API key values from read prompts written to .env |
| Internet -> container | git clone, apt, npm, uv all fetch from external sources |
| .env.example -> .env | Template file copied and modified with user-supplied values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | API key injection via read prompt | mitigate | Use sed with pipe delimiter `s\|...\|...\|` to avoid shell injection; keys are written to .env file only, not executed. Values are treated as literal strings by sed. |
| T-01-02 | Information Disclosure | API keys in .env readable by other users | accept | LXC container is single-tenant; .env file permissions default to root-only (0644). Phase 3 can tighten to 0600 if needed. |
| T-01-03 | Tampering | Supply chain (git clone from GitHub) | accept | OpenMontage is a known public repo; HTTPS enforced by git. Community-scripts pattern does not pin commits (git clone HEAD is standard). |
| T-01-04 | Tampering | npm/pip dependency supply chain | accept | Standard risk for all community-scripts installers; npm install and uv pip install use upstream registries over HTTPS. No additional mitigation beyond framework standard. |
| T-01-05 | Denial of Service | Network failure during install | mitigate | `network_check` function runs before any downloads; `catch_errors` trap surfaces failures clearly rather than silently continuing. |
</threat_model>
